# Gemini Project & Deployment Reference

This file serves as a reference for Gemini regarding the deployment infrastructure, database setup, and architecture of Mellow Belly.

## Infrastructure & Deployment
* **Frontend Hosting:** Vercel (automatically deployed on pushes to the `main` branch of the GitHub repository `FerchizV/mellow-belly`).
* **Database Hosting:** Supabase Cloud (online cloud-hosted instance, Project ID: `olvcjhynbllwollftxxa`).
* **Development Flow:** Developed via Lovable UI, exported to GitHub, and deployed to Vercel/Supabase. There is no active local development database or local Supabase CLI setup.

## Database & Schema Guidelines
* **Database Schema:** The source of truth for the database schema is the live Supabase Cloud database. A clean reference backup of this schema is maintained in [supabase/schema.sql](file:///c:/Users/ferch/Documents/Coding/Mellow-Belly/mellow-belly/supabase/schema.sql).
* **Migrations:** Local migration files (`supabase/migrations/*.sql`) have been cleared to avoid schema drift since database changes are managed directly via the online Supabase Dashboard. Do not generate or depend on local migration files unless explicitly requested.
* **Column Types:** Important identifiers (`id`, `place_id`, `user_id`) are stored as `TEXT` in the live database. BigInt is used for rating values.

## Important Backend / RPC Details
* **Reviews Feed (`get_reviews_feed`):** 
  * The `get_reviews_feed` RPC function must be executable by **both `anon` and `authenticated`** roles. This is because anonymous (logged-out) visitors of the Discover page need to see the public reviews feed.
  * The function dynamically handles privacy checks by masking notes for non-owners unless `is_public` is true, meaning it is secure to expose to `anon`.
  * Ensure any SQL adjustments to this function cast text identifiers to UUID inside the select statements to match TypeScript expectations (e.g. `r.id::uuid`, `r.place_id::uuid`, `r.user_id::uuid`).
